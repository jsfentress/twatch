# Activity Indicator

| Component | Activity Indicator |
| --- | --- |
| Purpose | Shows which tmux sessions have had recent output, as a live amber dot next to the session name. Gives the user a passive, always-on signal of "something just happened over there" without having to attach. The indicator must be responsive enough to feel live (sub-second after a poll) but stable enough not to flicker on every redraw. |
| File map with roles | `twatch.py:14` `AMBER_HOLD_S = 2.0` — how long the dot stays lit after activity is detected. `twatch.py:15` `FMT` includes `#{session_activity}` — tmux's monotonically-increasing activity timestamp per session. `twatch.py:297` `curses.init_pair(1, 214, -1)` — the amber color (xterm-256 color 214). `twatch.py:312` `prev_activity` — snapshot of last-seen activity values, initialized on TUI start. `twatch.py:313` `amber_until` — `{session_name: monotonic_deadline}`. `twatch.py:395-400` — per-tick diff: if `fresh.activity > prev`, set `amber_until[name] = now + AMBER_HOLD_S`, then update `prev`. `twatch.py:401-404` — prune `prev_activity`/`amber_until` for dead sessions. `twatch.py:197-200` (in `draw`) — renders `●` with color pair 1 if `now < amber_until[name]`, else dim. |
| Best Practices | **Use `time.monotonic()`, not `time.time()`** — the amber deadline must be immune to wall-clock jumps (NTP, suspend/resume). The loop already uses monotonic throughout; keep it that way. **`prev_activity` is seeded from the first `list_sessions` call** so existing sessions don't all light up amber on TUI startup. Do not reset it mid-loop. **Tmux `session_activity` is a counter, not a boolean** — compare strictly greater, not equal; equal means "no new output this tick." **Clean up `prev_activity` and `amber_until` for dead sessions** (`:401-404`) or you leak memory and, worse, a recreated session with the same name inherits stale state. **Do not derive activity from `attached`** — a user may be attached and idle, or detached and actively producing output; only `session_activity` is meaningful. **Color 214 is intentional** — not yellow, not red. Keep it; it reads as "attention" without feeling like an error. |
