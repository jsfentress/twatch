# Metadata Store

| Component | Metadata Store |
| --- | --- |
| Purpose | Persistent per-session metadata (title, team, notes, provenance, last_seen) keyed by tmux session name. tmux itself is stateless across restarts for these fields; the store is the source of truth for human-curated context. It must survive crashes, corruption, and long absences of a session without losing or silently mutating user data. |
| File map with roles | `twatch.py:16-19` — store paths (`~/.twatch/sessions.json`, `.bak`, `STALE_SECONDS = 30d`). `twatch.py:49-62` `load_store` — tolerant read, rotates corrupt files to `.bak`, returns `{}` on any failure. `twatch.py:65-69` `save_store` — atomic write via `.json.tmp` + `os.replace`. `twatch.py:72-88` `default_entry` / `ensure_entry` — schema defaults and forward-compatible field backfill via `setdefault`. `twatch.py:91-98` `cleanup_stale` — refreshes `last_seen` for alive sessions, prunes entries unseen for > `STALE_SECONDS`. |
| Best Practices | **Never write the store non-atomically** — always go through `save_store` (tmp + rename) so a crash mid-write cannot truncate the file. **Never raise on a malformed store** — `load_store` rotates to `.bak` and returns `{}`; the TUI should always be launchable even if state is corrupt. **Always funnel field access through `ensure_entry`** so new fields added later are backfilled on existing entries without a migration step. **Never delete a session's entry just because it's not currently running** — only `cleanup_stale` may prune, and only after `STALE_SECONDS`. A tmux session that dies for an hour must come back with its metadata intact. **Do not treat the store as authoritative for liveness** — tmux is. The store only holds metadata; `list_sessions` decides what exists. |
