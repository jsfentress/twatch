# Metadata Store

| Component | Metadata Store |
| --- | --- |
| Purpose | Persistent per-session metadata (title, notes, last_seen) keyed by tmux session name. tmux itself is stateless across restarts for these fields; the store is the source of truth for human-curated context. It must survive crashes, corruption, and long absences of a session without losing or silently mutating user data. |
| File map with roles | `twatch/store.py:9-12` ŌĆö store paths (`STORE_DIR = ~/.twatch`, `STORE_PATH = sessions.json`, `STORE_BAK = sessions.json.bak`) and `STALE_SECONDS = 30 * 24 * 3600`. `twatch/store.py:15-28` `load_store` ŌĆö tolerant read; if the file is missing returns `{}`, if it parses but isn't a dict or fails to parse it rotates to `.bak` and returns `{}`. `twatch/store.py:31-35` `save_store` ŌĆö `STORE_DIR.mkdir(parents=True, exist_ok=True)`, write to `sessions.json.tmp`, then `os.replace` onto `sessions.json` (atomic). `twatch/store.py:38-43` `default_entry` ŌĆö `{title: name, notes: "", last_seen: int(time.time())}`; the title defaults to the session name, not the empty string, so a fresh entry is immediately readable. `twatch/store.py:46-52` `ensure_entry` ŌĆö creates the entry if missing, otherwise `setdefault`s every key from `default_entry` so old store files gain new fields without a migration. `twatch/store.py:55-62` `cleanup_stale` ŌĆö bumps `last_seen` for any name in `alive_names`, then drops entries whose last_seen is older than `STALE_SECONDS`. |
| Best Practices | **Never write the store non-atomically** ŌĆö always go through `save_store` (tmp + rename) so a crash mid-write cannot truncate the file. `STORE_DIR` is auto-created on first save, so callers don't need to `mkdir` themselves. **Never raise on a malformed store** ŌĆö `load_store` rotates to `.bak` and returns `{}`; the TUI should always be launchable even if state is corrupt. **Always funnel field access through `ensure_entry`** so new fields added later are backfilled on existing entries without a migration step. **Never delete a session's entry just because it's not currently running** ŌĆö only `cleanup_stale` may prune, and only after `STALE_SECONDS`. A tmux session that dies for an hour must come back with its metadata intact. **Do not treat the store as authoritative for liveness** ŌĆö tmux is. The store only holds metadata; `list_sessions` decides what exists. **`default_entry` seeds `title` with the session name** ŌĆö code that renders titles can fall back to the session name, but persistence-wise the store is never blank for a known session. **Session name is the primary key** ŌĆö every store entry is keyed by its tmux session name, so a rename is a key change, not a field update. **TUI rename migrates the entry atomically** ŌĆö the rename action first calls `tmux rename-session`; only on success does it run `store[new_name] = store.pop(old_name)` and then `save_store`, so the entry's `title`/`notes`/`last_seen` travel with the session under its new key. **TUI rename rejects collisions** ŌĆö if the target name already belongs to another live session the rename is refused before any tmux or store mutation, so the migration step only ever runs after a successful `tmux rename-session`. **External renames are not tracked** ŌĆö a raw `tmux rename-session` invoked outside twatch is invisible to the store; the old key becomes orphaned (and is eventually pruned by `cleanup_stale` once it ages past `STALE_SECONDS`), while the next poll creates a fresh `default_entry` under the new name. |
