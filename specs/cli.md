# CLI

| Component | CLI |
| --- | --- |
| Purpose | Non-interactive entry points for scripting and for registering/creating sessions from inside shell startup, tmux hooks, or other tools. The CLI exists so twatch can be wired into a workflow without opening the TUI ‚Äî e.g. a project's `start.sh` runs `twatch create` to spin up a named, pre-registered session, or a shell rc file runs `twatch register` to tag the session it was launched inside. The same `main` is the package entry point (`twatch = "twatch.cli:main"` in `pyproject.toml`); with no subcommand it falls through to launch the Textual TUI. |
| File map with roles | `twatch/cli.py:66-102` `main` ‚Äî argparse setup and dispatch; with no subcommand, guards on `tmux_ok()` and then imports and runs `TwatchApp().run()` from `twatch.app`, wrapped in `try/except` so any TUI crash exits `1` instead of dumping a traceback. `twatch/cli.py:12-23` `cmd_list` ‚Äî tab-separated `name\ttitle\tattached|detached` per session (third column is the literal string `"attached"` or `"detached"`), sorted by name, suitable for `awk`/`cut`. `twatch/cli.py:26-45` `cmd_register` ‚Äî requires `$TMUX`; resolves the current session name via `tmux.current_session_name()` (which calls `tmux display-message`), applies `--title`/`--notes` overrides, saves the store. `twatch/cli.py:48-63` `cmd_create` ‚Äî creates a detached session via `tmux.new_session(name, cmd)` (which `claudify`s the command), applies `--title` to the store, saves. `twatch/cli.py:79` ‚Äî `command` is declared as `argparse.REMAINDER` so an optional `-- cmd...` tail is captured intact. `twatch/cli.py:88-89` ‚Äî inside `main`'s `create` branch, strips a leading `--` from `args.command` so `twatch create foo -- vim` works as a user would expect. |
| Best Practices | **Every CLI command short-circuits on `tmux_ok()`** before touching the store or subprocess ‚Äî exit code 2, message to stderr. Keep this guard; it's the difference between a helpful error and a Python traceback. **Exit codes are load-bearing**: `0` success, `2` precondition failure (no tmux, not in a session, tmux call failed), `1` TUI crash. Scripts rely on these ‚Äî do not collapse them. **`cmd_register` refuses to run outside `$TMUX`** ‚Äî there is no sensible "current session" otherwise. Do not try to guess. **`cmd_list` output is tab-separated and stable** ‚Äî it is a public contract for shell pipelines. Do not add columns in the middle, do not switch to JSON without a flag, and keep the third column as the literal `attached`/`detached` string. **The CLI and TUI share `store_mod.load_store`/`save_store`** and both mutate `~/.twatch/sessions.json`. Because `save_store` is atomic (tmp + rename), interleaved runs are safe, but do not hold the store open across long operations in either path. **`argparse.REMAINDER` for the create command's tail argument** is deliberate ‚Äî it lets users pass `-- vim +NERDTree` without argparse eating the flags. The manual `--` strip in `main` at `cli.py:88-89` is part of that contract; do not move it into `cmd_create` or `argparse.REMAINDER` will leave the literal `--` as the first token. **`TwatchApp` is imported lazily inside `main`** so the CLI subcommands don't pay the Textual import cost on every `twatch list` invocation. |
