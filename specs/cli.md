# CLI

| Component | CLI |
| --- | --- |
| Purpose | Non-interactive entry points for scripting and for registering/creating sessions from inside shell startup, tmux hooks, or other tools. The CLI exists so twatch can be wired into a workflow without opening the TUI — e.g. a project's `start.sh` runs `twatch create` to spin up a named, pre-registered session, or a shell rc file runs `twatch register` to tag the session it was launched inside. |
| File map with roles | `twatch.py:497-536` `main` — argparse setup and dispatch; also the fallback entry into `curses.wrapper(run_tui)` when no subcommand is given. `twatch.py:415-427` `cmd_list` — tab-separated `team\tname\ttitle\tstatus` per session, sorted by name, suitable for `awk`/`cut`. `twatch.py:430-458` `cmd_register` — requires `$TMUX`; resolves current session via tmux, applies `--title/--team/--notes` overrides, saves store. `twatch.py:461-481` `cmd_create` — creates a detached session, marks `created_by_twatch=True`, applies metadata, saves. Accepts an optional `-- command...` tail via `argparse.REMAINDER`. `twatch.py:484-494` `self_check` — hidden `--self-check` flag; smoke-tests `default_entry` and `cleanup_stale`. `twatch.py:524-525` — strips a leading `--` from `args.command` so `twatch create foo -- vim` works as a user would expect. |
| Best Practices | **Every CLI command short-circuits on `tmux_ok()`** before touching the store or subprocess — exit code 2, message to stderr. Keep this guard; it's the difference between a helpful error and a Python traceback. **Exit codes are load-bearing**: `0` success, `2` precondition failure (no tmux, not in a session), `1` TUI crash. Scripts rely on these — do not collapse them. **`cmd_register` refuses to run outside `$TMUX`** — there is no sensible "current session" otherwise. Do not try to guess. **`cmd_list` output is tab-separated and stable** — it is a public contract for shell pipelines. Do not add columns in the middle, do not switch to JSON without a flag. **The CLI and TUI share `load_store`/`save_store`** and both mutate `~/.twatch/sessions.json`. Because `save_store` is atomic (tmp + rename), interleaved runs are safe, but do not hold the store open across long operations in either path. **`argparse.REMAINDER` for the create command's tail argument** is deliberate — it lets users pass `-- vim +NERDTree` without argparse eating the flags. The manual `--` strip at `:524` is part of that contract. |
