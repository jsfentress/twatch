# TUI Shell

| Component | TUI Shell |
| --- | --- |
| Purpose | The curses event loop that renders sessions grouped by team, handles keybinds, and brokers all interactive actions (new, edit, kill, register, filter, attach). It is the user-facing surface of twatch; correctness here means the screen always reflects current tmux state within one poll interval, input is never lost, and leaving the TUI (attach or crash) never corrupts the terminal. |
| File map with roles | `twatch.py:13-14` `POLL_MS = 500`, `AMBER_HOLD_S = 2.0` ‚Äî loop cadence and activity decay. `twatch.py:116-133` `build_view` ‚Äî groups sessions by team, sorts, returns `(rows, selectable)` where `rows` is a mix of `("header", label)` and `("session", s, meta)` and `selectable` is the subset of row indices the cursor may land on. `twatch.py:136-158` `prompt` ‚Äî blocking inline bottom-line text input; Enter submits, Esc returns `None`. `twatch.py:161-163` `confirm` ‚Äî y/N wrapper over `prompt`. `twatch.py:166-171` `status` / set_status at `:319-322` ‚Äî transient bottom-line messages with a monotonic expiry. `twatch.py:174-211` `draw` ‚Äî full redraw; header, filter line, grouped rows, selection highlight, amber dot, status. `twatch.py:214-241` `HELP_TEXT` + `show_help` ‚Äî modal help screen. `twatch.py:244-260` `edit_metadata` ‚Äî three sequential prompts for title/team/notes. `twatch.py:263-282` `new_session`, `twatch.py:285-291` `kill_session` ‚Äî action helpers. `twatch.py:294-412` `run_tui` ‚Äî the main loop: init colors, load store, first poll, then `getch` dispatch + re-poll each tick. |
| Best Practices | **`run_tui` is the only place that owns the store in memory** ‚Äî all mutations happen through it and are followed by `save_store`. Action helpers receive `store` or `meta` but do not persist; the loop decides when to flush. **`stdscr.timeout(POLL_MS)` drives both input and polling** ‚Äî `getch` returns `-1` on timeout, which falls through to the re-poll block. Do not add a separate sleep. **`build_view` returns two parallel structures** (`rows` and `selectable`) because headers are non-selectable rows interleaved with sessions; `sel_idx` indexes into `selectable`, not `rows`. Do not collapse them. **Clamp `sel_idx` after every re-poll** (see `:408-412`) ‚Äî the session under the cursor may have died or been filtered out between ticks. **Esc is overloaded**: it clears an active filter first, and only quits if there is no filter. Preserve this ‚Äî it's the expected vim-ish behavior. **After `attach` returns, call `stdscr.clear(); stdscr.refresh()`** ‚Äî curses state is stale after tmux took the terminal. **Never raise out of `run_tui`** ‚Äî `main` wraps it, but a raised exception inside curses leaves the terminal in raw mode on some platforms. Prefer `set_status` for recoverable errors. **Do not block the loop on long tmux calls** ‚Äî all current tmux invocations are fast, but if you add one that isn't, move it off the main tick. |
