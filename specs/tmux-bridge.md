# Tmux Bridge

| Component | Tmux Bridge |
| --- | --- |
| Purpose | The single boundary between twatch and the `tmux` binary. Every read of live session state and every mutation (create, attach, kill, rename, capture) goes through here. Isolating tmux calls in one place means the rest of the code never has to think about `subprocess`, tmux format strings, or the difference between "tmux missing" and "tmux ran and returned nothing." |
| File map with roles | `twatch/tmux.py:10` `FMT` — the `list-sessions` format string `#{session_name}\|#{session_attached}\|#{session_activity}`. `twatch/tmux.py:13-14` `tmux_ok` — `shutil.which("tmux")` PATH probe; CLI entry points call this first. `twatch/tmux.py:17-18` `in_tmux` — `bool($TMUX)`; used by `attach()` and by the TUI to choose between `switch-client` and `tmux attach`. `twatch/tmux.py:21-35` `claudify` — if a command's argv[0] is `claude`, inserts `--dangerously-skip-permissions` after it (idempotent); applied automatically by `new_session`. `twatch/tmux.py:38-61` `list_sessions` — runs `tmux list-sessions -F FMT`, returns `None` if the binary is missing, `[]` on a non-zero exit, otherwise a list of `{name, attached: bool, activity: int, dead: bool}`. `twatch/tmux.py:64-81` `_dead_sessions` — private; runs `tmux list-panes -a -F "#{session_name}\|#{pane_dead}"` and returns the set of session names with at least one dead pane; `list_sessions` joins this in. `twatch/tmux.py:84-91` `attach` — when inside tmux, runs `tmux switch-client -t NAME` (capturing output so the caller can surface errors); when outside, runs `tmux attach -t NAME` and lets it inherit the terminal. `twatch/tmux.py:94-99` `new_session` — `tmux new-session -d -s NAME`; if `cmd` is non-empty, runs it through `claudify` and appends `/bin/sh -c <quoted>`. Used by both the TUI's `action_new_session` and the CLI's `cmd_create`. `twatch/tmux.py:102-106` `kill_session` — `tmux kill-session -t NAME`. `twatch/tmux.py:109-113` `rename_session` — `tmux rename-session -t OLD NEW`. `twatch/tmux.py:116-121` `capture_pane` — `tmux capture-pane -p -t NAME`, returns stdout or `""`. `twatch/tmux.py:124-132` `current_session_name` — `tmux display-message -p '#{session_name}'`; the canonical way to resolve "what session am I in?", used by `cmd_register`. |
| Best Practices | **`list_sessions` returns `None` vs `[]` deliberately** — `None` means tmux is unreachable (binary missing, fork failed), `[]` means tmux ran fine and there are simply no sessions. Callers must distinguish; the TUI bails out on `None` and renders an empty tree on `[]`. **`attach` is dual-mode by design** — inside tmux, `switch-client` keeps the user inside their existing client and never blocks; outside, `tmux attach` takes the terminal, so the TUI must yield it via Textual's `self.suspend()` first. Do not try to unify the two paths. **Never shell out with `shell=True`** — all tmux invocations pass an argv list so session names can't be shell-injected. The one place a string command goes through `/bin/sh -c` is `new_session`, and it is `shlex.quote`d after `claudify`. **Keep `FMT` pipe-delimited with a `split("\|", 2)`** — session names cannot contain `\|` in tmux, and the trailing fields are numeric, so a bounded split is safe and avoids regex. **`_dead_sessions` is best-effort** — if `list-panes` fails it returns an empty set, and `list_sessions` simply marks nothing dead rather than failing the whole listing. **`current_session_name` is the right way to resolve "current session"**, not parsing `$TMUX` — `$TMUX` only tells you that you're inside tmux, not which session. **`claudify` is idempotent** — it checks for an existing `--dangerously-skip-permissions` flag and is a no-op if found, so repeated wrapping is safe. |
